Syntax 

select * from table(
    applib.collapse_rows(
        cursor( SQL_QUERY ),
        DELIMITER
    )
);

Purpose

Given a table with two columns, where there is a 1-to-m mapping between the first column and the second column, for every distinct value n in the first column, concatenate all values which map to it in the 2nd column. The table input must be ordered by the first column, which must also be of type VARCHAR or CHAR.

Parameters

Input:

  • SQL_QUERY: sql query for the cursor input; input should have 2 columns, where the first column corresponds to the parent_value or key column, and the second column corresponds to the child_value or values column. The first column must be of type VARCHAR or CHAR, and the sql query must be sorted in ascending order by the first column. Additional ordering on the second column can be used to control concatenation sequence
  • DELIMITER [VARCHAR(1)]: character used to separate child values within the concatenation

Output:

  • table which contains the following columns. If input table has no rows, output table will return no rows
    • PARENT_VALUE [VARCHAR(65535)]: corresponds to the first column in the cursor input
    • CONCATENATED_CHILD_VALUES [VARCHAR(65535)]: values in the second column which map to a particular PARENT_VALUE, concatenated with DELIMITER as the delimiter value. *** If adding a child value would exceed the maximum concatenation length of 16384, then it and subsequent child values will not be added.
      • If the first child value exceeds the maximum concatenation length, then CONCATENATED_CHILD_VALUES is set to empty string
      • IF child value is null and there is only one child, CONCATENATED_CHILD_VALUES is set to null. If there are other child values, the null value is ignored.
    • COLLAPSED_ROW_COUNT [INTEGER]: Number of child values which were added to the CONCATENATED_CHILD_VALUES. Null valued children do not increase the COLLAPSED_ROW_COUNT.

Exceptions:

A fatal exception is thrown under any of the following conditions:

  • Incorrect number of input/output columns
  • First column in input cursor is not of type VARCHAR or CHAR
  • Input table is not presorted correctly
